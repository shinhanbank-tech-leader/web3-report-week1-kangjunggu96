# 제목: 블록체인의 신뢰 형성 구조와 기존 금융 시스템의 신뢰 구조 비교

## 부서명: 디지털서비스개발부
## 이름: 강정구

-------------------

## 레포트 내용


### 1. 개요

시장은 거래 상대방이 약속을 이행할 것이라는 '신뢰'가 없다면 성립할 수 없습니다.
전통적인 금융 시스템은 중앙집중화된 기관을 통해 신뢰를 구축합니다.
그에 반해, 블록체인은 수학적 알고리즘과 네트워크를 통해 신뢰를 기술적으로 구현했습니다.
이 두 시스템의 신뢰 형성 원리를 비교하고 특징을 비교해 보았습니다.

### 2. 기존 금융 시스템

기존 금융시스템은 중앙 집중형 구조를 기반으로 합니다.

- 신뢰의 주체 : 은행, 정부, 중앙은행 등 제 3의 공인기관
- 신뢰 형성 방식
    - 법적, 제도적 장치 : 국가가 제정한 법률과 규제(금감원, 예금자보호법 등) 을 통해 시스템의 안전성 보장
        - 금융실명제 : 자금세탁 방지, 거래의 투명성 확보
        - 예금자보호제도 : 금융기관 파산하더라도 예금보험공사를 통해 제한 금액까지 보장
        - 금융소비자보호법(금소법) : 판매 원칙을 위반한 금융사에 강력한 징벌적 과징금 부과, 소비자의 청약 철회권 강화 등
    - 기관의 평판 : 오랜 역사와 자본력을 가진 금융기관의 사회적 신용을 기반
        - 금융위원회 : 금융 정책 수립, 금융기관의 진입/퇴출을 담당
        - 금융감독원 : 금융기관의 건전성 직접 검사, 소비자 분쟁 조정
        - 한국은행 : 통화 정책을 통해 물가 & 금융시스템 안정 책임
    - 중개자의 역할 : 거래 당사자 사이에 은행이 개입하여 장부기록, 송금 보증
        - 금융결제원 기반 망
        - 공동인증 및 보안체계
        - 오픈뱅킹 및 마이데이터
- 한계
    - 중개 비용 : 거래마다 수수료 발생, 시스템 유지비용 큼
    - 단일 실패 지점 : 중앙서버의 해킹문제, 기관의 부패 문제등으로 시스템이 마비되거나 신뢰 붕괴 가능
    - 폐쇄성 : 장부가 기관 내부에서만 관리되므로 거래 과정 투명성이 낮음 


### 3. 블록체인 신뢰형성 구조

블록체인은 탈중앙화 분산 구조를 기반으로 합니다.

   * 신뢰의 주체: 코드, 수학적 알고리즘, 네트워크 참여자 전체.
   * 신뢰 형성 방식
       * 합의 알고리즘(Consensus Algorithm): 작업 증명(PoW)이나 지분 증명(PoS) 등을 통해 네트워크
         참여자들이 거래의 유효성을 공동으로 검증
       * 분산 원장 기술(DLT): 모든 거래 내역을 네트워크 참여자 모두가 공유하므로, 특정인이 데이터를
         조작하는 것이 사실상 불가능
       * 암호화(Cryptography): 해시 함수와 디지털 서명을 통해 데이터의 무결성과 보안을 확보합니다.
       * 스마트 컨트랙트(Smart Contract): 조건이 충족되면 계약이 자동으로 실행되는 코드를 통해 인간의
         개입 없이도 계약 이행을 보장
   * 특징
       * 무신뢰성(Trustless): 상대방을 믿지 않아도 시스템(코드)을 믿음으로써 거래가 가능해집니다.
       * 투명성: 누구나 원장 내역을 확인할 수 있어 불투명한 거래를 방지합니다.
       * 검열 저항성: 특정 권력 기관이 거래를 임의로 중단하거나 취소할 수 없습니다.
    * 한계
       * 가계부채 문제 : DeFi(탈중앙화 금융)을 통해 누구나 쉽게 담보대출을 받을 수 있게 되면 오히려 레버리지가 극대화되어 부채리스크의 극대화 발생 가능
       * 금융 사고 및 보안 위협 : 보이스피싱의 케이스를 생각해봤을때 블록체인은 거래의 취소 자체가 불가능. 사기 계좌의 동결이나 송금 지연할 수 있는 안전장치가 순수 블록체인엔 존재하지 않음
       * 디지털 격차 : 현재 뱅킹 앱도 어려운 고객에게 '개인키 관리', '시드 구문 보관' '가스비 계산'은 넘기 어려운 벽


### 4. 두 시스템의 결합

    * 한계를 넘기위한 대안들
        * 프로그래밍이 가능한 돈 (CBDC) : 정부가 발행하는 디지털 화폐에 용도제한, 사후추적 등의 기능을 넣어 보이스피싱 자금 동결 등의 기술 연구중
        * 탈중앙화 신원증명 (DID) : 복잡한 인증 절차없이 지문 하나로 안전하게 본인을 증명하여 비교적 간단하게 보안 사고 방지
        * RWA (Real World Asset) 토크나이징 : 현실자산(채권, 금, 부동산 등)을 토큰 형태로 발행하여 블록체인 위에서 거래하도록 함
        * 송금 & 결제망의 탈중개화 : 기존의 느리고 비싼 SWIFT를 블록체인으로 대체하려는 시도
        
    => 결론적으로 기술은 도구일 뿐 해결책이 아님을 아는것이 중요합니다.
    대한민국 금융시스템의 한계점은 기술의 부족함 때문이 아니라 제도의 사각지대, 개인의 탐욕, 사회적 구조 불균형에서 기인한 것들이 많습니다.
    블록체인은 신뢰를 확인하는 비용을 낮춰주는 용도로 사용은 가능하지만, 기존에 고객들이 직면한 근본적인 문제들은 기술만으로 해결하기엔 한계가 있습니다.
